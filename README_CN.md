## 为什么需要这个项目？

### 当前 AI 的根本问题

无论是 AI 编程工具（Cursor、Copilot）、AI 聊天软件（ChatGPT、Claude）还是 AI 角色扮演平台（SillyTavern），都面临相同的底层限制：

| 问题               | 现状                                   | 后果                                    |
| ------------------ | -------------------------------------- | --------------------------------------- |
| **上下文窗口有限** | 即使 128K-1M tokens，长对话仍会溢出    | 早期对话被截断，AI 丢失关键信息         |
| **注意力分散**     | 上下文越长，模型对每段内容的关注度越低 | 即使信息在上下文中，AI 也可能"视而不见" |
| **无持久记忆**     | 关闭对话 = 遗忘一切                    | 每次新对话都从零开始                    |

### 我们的解决方案

**不把所有记忆塞进上下文，而是让专门的 AI 按需检索。**

```
传统方案：[全部历史记忆 + 当前对话] → 单个 AI → 注意力分散
                    ↓
我们的方案：[索引] → 检索 AI（专注找记忆）→ [精选记忆 + 当前对话] → 回复 AI（专注回复）
```

回复 AI 只看到经过检索 AI 筛选后的**精准记忆片段**，上下文干净、信噪比极高，注意力**完全不会发散**。

## 核心特性

### 🧠 原创分层记忆算法

参照人类大脑海马体的记忆形成机制与艾宾浩斯遗忘曲线设计，实现**理论上无限期的 AI 记忆**。

#### 三层记忆架构

```
🔥 热记忆层 (Hot) — 每轮自动注入
   用户画像 / 永久记忆 Top-100 / 未完成任务 / 近期关于用户的记忆

🌤️ 温记忆层 (Warm) — 按需检索，最近 1 个月
   每日总结 / 临时记忆归档 / 月度索引

❄️ 冷记忆层 (Cold) — 深度检索，超过 1 个月
   月度总结 / 历史日总结 / 年份索引
```

此外还有 **L0 记忆表格层**（10 张高度自定义表格，每轮全量注入），提供结构化的即时上下文。

#### 关键数据

| 指标           | 数值                                                       |
| -------------- | ---------------------------------------------------------- |
| 热层每轮注入   | **~7,000-11,000 tokens**（128K 窗口仅占 5-9%）             |
| 检索 AI 上下文 | **<5,000 tokens**（100% 注意力集中于检索任务）             |
| P1 检索效率    | **最多 3 轮**即命中目标（BM25 预筛选 + 正则精确匹配）      |
| 检索技术栈     | **BM25 + 正则搜索**（双引擎协作，零外部依赖）              |
| 存储成本       | **零**（纯 JSON 文件，无数据库依赖）                       |
| 单角色持续运行 | **12+ 年**（按 5,000 文件计算）                            |
| 理论持续时间   | **260+ 年**（按 100,000 文件计算，NTFS/ext4 支持远超此数） |

#### 记忆衰减公式

```
score = weight × (1 / (1 + days_since_triggered × 0.1))
```

借鉴艾宾浩斯遗忘曲线，重要且近期触发的记忆优先注入，而非简单的时间顺序。

#### 纯提示词驱动 — 零硬编码限制

记忆系统最关键的设计特点：**所有记忆的注入、提取、归档、总结操作全部由 AI 通过提示词完成**，而非传统的硬编码逻辑。

这意味着：

- **记忆表格的含义和用途随时可改**：只需修改提示词中对表格的描述，AI 就会按新的方式理解和操作表格，无需改一行代码
- **归档策略可随时调整**：P2-P6 的行为完全由提示词定义，修改提示词即可改变归档规则、总结格式、检索策略
- **零技术门槛迁移**：用户可以自行编辑提示词来适配不同场景（角色扮演 / 编程助手 / 游戏 NPC），无需编程能力
- **天然避免技术债务**：没有复杂的解析器或状态机需要维护，AI 本身就是最灵活的"解析器"

### 📊 高度自定义的记忆表格 — 完美解决 AI RP 上帝视角问题

**10 张完全可自定义的结构化表格**，每轮以 CSV 格式注入上下文。表格的含义和用途**完全由提示词定义**——修改提示词描述，就能让同一套表格系统服务于完全不同的场景：

| 场景            | 表格用法示例                                                                                           |
| --------------- | ------------------------------------------------------------------------------------------------------ |
| **AI 角色扮演** | 时空设定 / 角色状态 / 社交关系 / 任务进度 / 物品库存 — **AI 只知道表格中记录的信息，完美解决上帝视角** |
| **编程项目**    | 架构决策 / 代码约定 / 模块依赖 / Bug 跟踪 / TODO 列表                                                  |
| **工作管理**    | 项目进度 / 会议记录 / 联系人 / 待办事项 / 知识积累                                                     |
| **游戏**        | 角色属性 / 装备列表 / 技能树 / 世界状态 / NPC 关系                                                     |

表格内容由聊天 AI 通过 `<tableEdit>` 标签自动维护——AI 在对话中自主决定何时更新哪些数据，无需用户手动操作。

**为什么这能解决上帝视角？** 在传统 AI RP 中，AI 能"看到"所有对话历史，包括角色本不该知道的信息。通过表格系统，AI 只根据表格中明确记录的信息来行动——如果某件事没有被记录在角色的认知表格中，AI 就不会"知道"它。这是一种**信息隔离机制**，让 AI 的行为更真实、更沉浸。

### 🌍 世界书动态注入 — 读取表格数据实现条件触发

世界书条目支持**动态注入**——根据记忆表格中的实时数据，决定是否注入某条世界设定：

```
世界书条目触发条件可以读取表格中的数值和状态：
  → 当表格#1"角色状态"中好感度 > 80 时，注入"特殊对话解锁"设定
  → 当表格#4"任务进度"中主线 = "第三章" 时，注入对应的世界观描述
  → 当表格#5"物品库存"中拥有某道具时，注入道具的使用效果说明
```

这种**表格驱动的动态世界构建**机制，简单易上手，让世界设定不再是静态文本，而是随着对话进程和角色状态实时变化的活系统。

### 🔍 智能检索系统 — BM25 + 正则搜索双引擎

**最新引入的双引擎检索能力**，让 AI 快速精准地找到需要的信息：

#### BM25 语义检索

- **经典高效**：基于词频-逆文档频率（TF-IDF）的统计算法，快速从海量记忆文件中筛选出最相关的候选
- **纯 JS 实现**：零外部依赖，不需要向量数据库或 Embedding API，即装即用

#### 正则表达式精确搜索

- **精确匹配**：在 BM25 粗筛后，使用正则表达式进行精确定位——支持模式匹配、关键词组合、模糊搜索
- **跨文件搜索**：一次搜索即可扫描所有记忆文件和项目文件，快速定位目标内容
- **IDE 深度集成**：在文件编辑环境中也可直接使用正则搜索，快速查找和定位项目中的任意文件和内容

#### 检索效果

- **P1 检索 AI 大幅增强**：BM25 预筛选 + 正则精确匹配的双引擎协作，检索轮次从原来的 **5 次优化为最多 3 次**即可精准命中目标记忆——速度提升 40%、API 成本降低 40%
- **检索准确率显著提高**：正则搜索弥补了纯语义检索在精确匹配上的不足，关键词、日期、名称等结构化信息可以一次命中

### 🤖 多 AI 协作引擎 — 成本可控的智能体系

系统内置 **7 个 AI 角色**，各司其职。**每次对话实际只调用 2 个 AI**（检索 AI + 对话 AI），其余 AI 按需触发，完全不用担心用量问题：

| AI           | 职责                                                | 触发方式                   | 用量说明                        |
| ------------ | --------------------------------------------------- | -------------------------- | ------------------------------- |
| 聊天 AI      | 与用户对话、文件操作                                | 用户发送消息               | 每次对话必调                    |
| P1 检索 AI   | 从记忆层检索相关历史（最多 3 轮）+ **智能预设切换** | 每轮自动触发               | 每次对话必调，**可使用免费 AI** |
| P2 归档 AI   | 临时记忆超阈值时总结归档                            | **约 50 次对话才触发一次** | 极低频，几乎无用量              |
| P3 日总结 AI | 生成当日详细总结                                    | **手动触发**               | 用户主动按才调用                |
| P4 热→温 AI  | 将过期热层记忆转入温层                              | **手动触发**               | 用户主动按才调用                |
| P5 月总结 AI | 温→冷归档，生成月度总结                             | **手动触发**               | 用户主动按才调用                |
| P6 修复 AI   | 检查并修复记忆文件格式                              | **手动触发**               | 用户主动按才调用                |

#### 💰 AI 用量与成本分析

**核心洞察**：检索 AI（P1）的任务是"找记忆"，不需要高智能模型，**完全可以使用免费或极低成本的 AI**（如 Gemini 2.0/2.5 Flash 免费额度）。

```
每次对话的 AI 调用：
  ① P1 检索 AI — 找记忆 + 判断预设切换（可用免费 AI，如 Gemini Flash）
  ② 聊天 AI   — 基于精选记忆生成回复（用你喜欢的任意模型）

不常用的 AI：
  ③ P2 归档 — 约 50 次对话才触发一次，几乎感知不到
  ④ P3-P6   — 全部手动触发，不按就不调用，零用量
```

**结论**：如果 P1 使用免费 AI（Gemini Flash 免费额度完全够用），那么每次对话的实际成本 = **只有聊天 AI 的一次调用**。记忆系统几乎是零成本运行的。

### 🔄 智能预设切换 — AI 自动适配交互模式

**重大突破**：P1 检索 AI 不仅检索记忆，还能**实时分析对话意图，自动切换最适合的提示词预设**。

- **多模式自适应**：日常闲聊、角色扮演、编程工作、提示词编写……AI 根据对话内容自动切换到最合适的预设，提示词和 COT（思维链）随之改变
- **无缝体验**：用户无需手动操作，说一句"帮我写代码"，AI 的行为模式就会实时调整
- **冷却防振荡**：内置冷却计数器，避免短时间内反复切换
- **完全可自定义**：切换逻辑由提示词中的 COT 指导，用户可自定义切换条件和策略
- **手动一键切换**：也支持在聊天界面手动快速切换预设

这使得 AI 不再是"一种预设走天下"，而是**根据当下场景动态适配最优行为模式**——真正的多模式智能体。

## 对未来 LLM 的思考

即使未来上下文窗口扩展到 **10M+ tokens**，分层记忆仍然有价值：

1. **注意力问题不会消失**：窗口再大，模型对海量文本的关注度仍然会下降。预筛选 + 精准注入永远优于"全部塞进去"。
2. **成本效率**：大窗口 = 高成本。用 ~10K tokens 的精选记忆替代 100K+ tokens 的全量历史，API 调用成本降低 **10 倍以上**。
3. **结构化 > 非结构化**：表格化的记忆比散落在对话中的信息更容易被 AI 准确读取和更新。

分层记忆不是上下文窗口的临时替代品，而是一种**更优的信息组织范式**。

## 快速开始

### 环境要求

- [Deno](https://deno.land/) 运行时
- 现代浏览器（Chrome / Edge / Firefox）
- 至少一个 AI API Key（推荐 Gemini API，免费可用）

### 安装与启动

```bash
# 克隆项目
git clone https://github.com/beilusaiying/always-accompany.git
cd always-accompany

# 启动（Windows）
run.bat

# 启动（Linux/macOS）
chmod +x run.sh
./run.sh
```

启动后浏览器访问 `http://localhost:1314`

### 基础配置

1. **配置 AI 源**：首页 → 系统设置 → 添加 AI 服务源（proxy / gemini 等）
2. **导入角色卡**：首页 → 使用 → 导入（支持 SillyTavern PNG/JSON 格式）
3. **配置记忆预设**：首页 → 记忆预设 → 为 P1-P6 配置 API（推荐 P1 使用 Gemini Flash 免费额度）
4. **开始对话**：点击角色卡进入聊天界面

### 记忆系统使用

- **自动运行**：记忆表格由聊天 AI 自动维护（通过 `<tableEdit>` 标签），检索 AI (P1) 每轮自动触发
- **手动操作**：聊天界面右栏 → 记忆 AI 操作 → P2-P6 手动按钮
- **日终归档**：每天结束时点击「结束今天」按钮，触发日终 9 步归档流程
- **记忆浏览**：聊天界面 → 记忆 Tab → 浏览/编辑/导入/导出记忆文件

### Discord Bot 配置

1. 在 [Discord Developer Portal](https://discord.com/developers/applications) 创建 Bot 应用
2. 在 beilu-chat 聊天界面顶部 Bot 标签页 → 填入 Bot Token 和 Owner 用户名
3. 点击"启动 Bot"→ 在 Discord 频道 @你的 Bot 即可对话
